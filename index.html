Design Rationale

For this portfolio I wanted to make the website from scratch. I started creating webpages when I realized I didn’t want to copy the header and footer into each individual file. So, I created a tool in Python to do this automatically. After creating the site further, I added more and more features to my Python tool to tailor my needs. I wanted my website to be simple but elegant. I used visually simple features on my site, not straying too far from the bootstrap defaults.

I used simple navigation where every page can be accessed from the header navigation bar. I made sure to use C.R.A.P. design principles when creating my design. I appropriately used contrast to make all elements visible to the user. Having a simple design helped with this process since most of the colors are black, white, and grey.

My main motivation was to see how capable I was of creating a website from scratch and making a tool to assist me. I was also curious to see how my skills and projects look when laid out in such a nice fashion. I wanted my portfolio to stand out amongst other potential candidates for jobs I’m interested in.
